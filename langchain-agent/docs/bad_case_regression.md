# Bad Case 回归表（固定样例）

说明：
- 该表用于对比“优化前 / 优化后”的稳定回归结果。
- `before` 与 `after` 分别记录一次同配置评测结果，便于面试中展示迭代闭环。
- 建议每次 Agent / RAG 策略变更后，至少回放这 10 条样例。

| case_id | query | 预期行为 | before | after | 结论 |
|---|---|---|---|---|---|
| BC-01 | 本周流量下滑，先排查什么 | 先分析流量再给行动建议 | 待填 | 待填 | 待填 |
| BC-02 | ROI低于1.8，预算怎么调 | 给出预算倾斜与暂停低效位建议 | 待填 | 待填 | 待填 |

## 推荐打分口径
- `准确性`：结论是否与工具证据一致。
- `完整性`：是否覆盖问题摘要 / 根因 / 行动计划 / 风险跟进。
- `可执行性`：是否给出明确可执行动作。
- `鲁棒性`：异常输入、注入指令、缺字段场景是否稳定。

打分建议：每条 0~2 分，总分 8 分，`after >= before` 视为通过。
